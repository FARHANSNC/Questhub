=========================================
QUESTHUB - CODEBASE FUNCTIONALITY SUMMARY
=========================================

This document explains the purpose and functionality of the main files, models, and features across the entire QuestHub codebase. Use this as a reference guide to explain the project architecture and features to your teacher.

--------------------------------------------------------------------------------
1. BACKEND (Node.js + Express + MongoDB + Socket.IO)
--------------------------------------------------------------------------------

[ MODELS - Database Schemas built with Mongoose ]
- User.model.js
  Stores user information (name, email, password) and handles authentication logic (password hashing, role management).
- Quiz.model.js
  Represents a single quiz entity, including its title, description, time limit, difficulty, and references to the subject it belongs to.
- Question.model.js
  Stores individual quiz questions with multiple options and marks which option is correct. Has a reference to its parent Quiz.
- Subject.model.js
  Represents categories or topics (e.g., Science, History) that group multiple quizzes together.
- Result.model.js
  Stores a user's performance on a completed quiz, including their score, time taken, list of correct/incorrect answers, and pass/fail status.
- Room.model.js
  Manages multiplayer live sessions (Group Play). Stores the unique room code, host user, current quiz being played, and lists of players in the lobby.

[ ROUTES - API Endpoints for Frontend Communication ]
- auth.routes.js
  Handles user registration, login, and token generation (JWT).
- user.routes.js
  Manages the user's profile updates and fetching user-specific data.
- quiz.routes.js
  Performs CRUD (Create, Read, Update, Delete) operations on quizzes. Allows users to create their own quizzes and retrieve random or subject-specific quizzes.
- question.routes.js
  Handles adding, editing, and deleting questions for a specific quiz.
- subject.routes.js
  Manages subject categories (fetching all subjects or creating new ones).
- result.routes.js
  Saves a completed quiz result, fetches a specific result by ID, and retrieves a user's entire history of results.
- room.routes.js
  API endpoints for creating a live multiplayer room and returning room connection details.
- analytics.routes.js
  Provides aggregate statistics for the dashboard (e.g., total users, quizzes created, total attempts).

[ REAL-TIME SOCKET SERVER - Multiplayer Buzzer Features ]
- socket/index.js (or similar)
  Handles real-time WebSocket events using Socket.IO.
  Tasks include:
  * user joining/leaving rooms
  * broadcasting real-time room states
  * host starting the game
  * sending questions to all connected players simultaneously
  * determining the winner of the "Buzzer" (first person to click)
  * broadcasting answer accuracy and live leaderboard updates.

--------------------------------------------------------------------------------
2. FRONTEND (React.js + Vite)
--------------------------------------------------------------------------------

[ CONTEXT & SERVICES ]
- AuthContext.js
  Global state manager that keeps track of the currently logged-in user and manages the JWT token via localStorage.
- api.js
  Axios interceptor setup. It automatically attaches the user's JWT token to every outgoing API request to ensure secure communication with the backend.

[ CORE PAGES & COMPONENTS ]
- Home.jsx
  The landing page. Explains the platform's features and prompts users to login or register.
- Login.jsx / Register.jsx
  Authentication pages. Handles user inputs and communicates with `auth.routes.js` to securely log users into the system.
- Dashboard.jsx
  The main user hub. Displays statistics, quick actions to play quizzes, and a "My Quizzes" tab where logged-in users can create, edit, or delete their own custom quizzes.
- AdminDashboard.jsx
  A privileged interface for managing the platform. Allows administrators to view overall platform metrics and delete any user's quizzes to moderate content.
- RandomQuiz.jsx
  Setup screen for generating a quick quiz tailored by length and difficulty.
- SubjectQuiz.jsx
  A browsing interface for picking quizzes based on specific subjects/topics.
- QuizAttempt.jsx
  The primary single-player gameplay interface. It fetches questions, manages a real-time countdown timer, records selected options, and submits the final grade.
- GroupPlay.jsx
  The multiplayer lobby and gameplay page. Uses Socket.IO to manage a real-time game where players join via a 'Room Code', race to click a buzzer when a question appears, and compete on a live leaderboard.
- Results.jsx
  Displays the outcome of a single quiz attempt. Shows score percentages, time taken, and provides a question-by-question breakdown of what the user got right or wrong.
- History.jsx
  A timeline view of the user's past quiz attempts and overall lifetime statistics.

--------------------------------------------------------------------------------
SUMMARY OF WORKFLOWS (Example for your presentation)
--------------------------------------------------------------------------------
1. Single Player Flow:
   User logs in -> Goes to Dashboard -> Clicks Random Quiz -> Plays QuizAttempt (timer ticks) -> Submits -> Redirected to Results page displaying score.
2. Group Play Flow (Buzzer Mode):
   Host creates a room via GroupPlay -> Generates a Room Code.
   Friends join using the Room Code -> Host clicks "Start".
   Backend sends questions via WebSockets -> First to buzz locks out others -> They answer -> Results sync live on Leaderboard.
3. Quiz Creation Flow:
   User (or Admin) goes to Dashboard "My Quizzes" -> Clicks "Create Quiz" -> Defines questions/answers -> Saved to database -> Ready to be played by anyone.

NOTE: The entire app interface uses a centralized, custom-built CSS Design System (index.css) utilizing CSS variables to enforce a pure monochrome (black, white, and grayscale glassmorphism) theme throughout all components.
